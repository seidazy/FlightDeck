# FlightDeck — Документация фронтенда

## Обзор

Радар полётов в реальном времени. React + TypeScript + Leaflet. Подключается к WebSocket бэкенду и отображает самолёты на интерактивной карте с тёмной темой.

## Стек технологий

| Технология | Назначение |
|---|---|
| React 19 | UI фреймворк |
| TypeScript | Типизация |
| Vite | Сборщик и dev-сервер |
| Leaflet + react-leaflet | Рендеринг карты |
| WebSocket (нативный) | Получение данных в реальном времени |

## Архитектурные решения

### Почему "group by type" а не FSD?

Проект — одна страница, одна фича, ~8 файлов. FSD добавил бы пустые слои без пользы. Group-by-type — правильный выбор для такого масштаба. При росте проекта (фильтры, история, авторизация) можно мигрировать на FSD.

### Почему сырые Leaflet маркеры вместо `<Marker>` из react-leaflet?

Нужен императивный контроль для перепозиционирования маркеров при скролле через копии мира (бесконечный горизонтальный скролл). Декларативный `<Marker>` из react-leaflet это не поддерживает.

### Почему heading считается на фронтенде?

Эндпоинт `/ws/planes/basic` не возвращает heading. Мы вычисляем его из дельты позиций между последовательными сообщениями WebSocket через `atan2(deltaLng, deltaLat)`.

### Почему самолёты не рендерятся до 2-го сообщения WebSocket?

При первом сообщении нет предыдущей позиции для сравнения — heading был бы 0° (все самолёты смотрят вверх). Ждём второе обновление чтобы получить реальный heading.

## Функциональность

### Основное
- Позиции самолётов в реальном времени на тёмной карте (CARTO dark tiles)
- Иконки самолётов вращаются по вычисленному heading
- Клик по самолёту → открывается панель деталей, карта летит к нему
- Повторный клик или Escape → снимает выделение

### Панель деталей
- Номер рейса, авиакомпания, статус (цветной бейдж)
- Маршрут: аэропорт вылета → аэропорт прилёта
- Сетка данных: модель, регистрация, высота, скорость, курс, вертикальная скорость, длительность, ETA
- Полоса загруженности пассажиров
- Доступность с клавиатуры (Escape для закрытия, фокус на панели)

### Устойчивость
- Авто-переподключение при обрыве WebSocket (задержка 2с)
- Индикатор статуса соединения в шапке (зелёный/жёлтый/красный)
- Обработка ошибок для некорректных WS сообщений (try/catch на JSON.parse)
- Спиннер загрузки до получения первых данных

### Поведение карты
- Бесконечный горизонтальный скролл (мир повторяется)
- Самолёты перепозиционируются на видимую копию мира при `moveend`
- Вертикальные границы ограничены (нет белых полос)
- Тултип при наведении с ID самолёта

## Поток данных

```
Backend WS ──→ usePlanesBasic() ──→ App (state + расчёт heading) ──→ Map ──→ PlaneMarker
                                                                            ↕
Backend WS ──→ usePlaneDetails() ──→ App ──→ DetailPanel
```

1. `usePlanesBasic` подключается к `/ws/planes/basic`, получает все самолёты каждую ~1с
2. App вычисляет heading сравнивая текущую и предыдущую позиции
3. Map рендерит PlaneMarker только для самолётов с вычисленным heading
4. При клике устанавливается `selectedId` → `usePlaneDetails` открывает новый WS к `/ws/planes/details` и подписывается
5. DetailPanel рендерит детальные данные

## Ключевые детали реализации

### Перенос долготы (`wrapLng`)

```typescript
// Сдвигает долготу в диапазон ±180° от точки отсчёта
function wrapLng(lng: number, centerLng: number): number {
  while (lng - centerLng > 180) lng -= 360;
  while (lng - centerLng < -180) lng += 360;
  return lng;
}
```

Используется в PlaneMarker чтобы маркер всегда был на видимой копии мира.

### Вычисление курса (`computeHeading`)

```typescript
function computeHeading(prevLat, prevLng, currLat, currLng): number | null {
  const dLat = currLat - prevLat;
  const dLng = currLng - prevLng;
  if (Math.abs(dLat) < 0.0001 && Math.abs(dLng) < 0.0001) return null;
  return (Math.atan2(dLng, dLat) * 180) / Math.PI;
}
```

Возвращает градусы: 0° = север, 90° = восток. Возвращает null если движение слишком маленькое (избегаем дёрганья).

### Стабильный обработчик клика (паттерн ref)

```typescript
const onClickRef = useRef(onClick);
onClickRef.current = onClick;
// ...
marker.on("click", () => onClickRef.current());
```

Избегаем перепривязки event listener при изменении пропа `onClick`, при этом всегда вызываем актуальную функцию.
